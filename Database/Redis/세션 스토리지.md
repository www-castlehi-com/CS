## 1. 세션
로그인 기능에는 JWT 방식과 세션 방식 등이 있다.
세션 방식은 다음과 같은 흐름으로 동작한다.
1. 사용자가 ID / 비밀번호로 로그인을 요청한다.
2. 서버는 인증이 성공하면 세션 ID를 생성한 뒤, 해당 세션 ID에 사용자 정보를 매핑하여 세션 저장소에 저장한다.
3. 서버는 응답으로 세션 ID를 쿠키에 담아 클라이언트에 전달한다.
4. 클라이언트는 요청마다 쿠키에 포함된 세션 ID를 함께 전송한다.
5. 서버는 세션 ID를 기준으로 세션 저장소에서 사용자 정보를 조회해서 로그인을 처리한다.
## 2. 서버가 2대 이상일 때, 로컬 세션 스토리지
![](https://i.imgur.com/2iQw99p.png)
로컬 세션 스토리지를 사용할 때, 서버가 2대 이상이라면 각 서버가 가지고 있는 세션 정보를 서버들끼리 공유하지 않아 문제가 발생한다.
한 명의 사용자가 로드밸런서에 의해 여러 서버에 요청을 골고루 보내게 되는데, 어떤 요청에서는 인증이 성공하지만 어떤 요청에서는 인증이 실패할 수 있다.
![](https://i.imgur.com/KWwAqRB.png)
따라서 로컬 서버와 별개인 외부 세션 스토리지가 필요하다.
외부 세션 스토리지를 사용하게 되면 세션 정보를 외부에 저장하기 때문에, 각기 다른 서버에 요청이 되어도 세션 정보를 공유할 수 있게 된다.

이 때 외부 세션 스토리지로 Redis를 많이 사용한다.
