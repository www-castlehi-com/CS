위의 예시에서는 상속 관계는 아니지만, 테이블마다 공통적인 필드가 반복적으로 등장한다.
이 때 모든 테이블에 공통적인 속성을 필드로 가질 경우, 메모리 낭비 및 조회 성능이 저하될 수 있다.
## `@MappedSuperclass
```java
@MappedSuperclass  
public abstract class BaseEntity {  
  
    private String createdBy;  
    private LocalDateTime createdDate;  
    private String lastModifiedBy;  
    private LocalDateTime lastModifiedDate;
}
```
공통 속성을 모아둔 클래스에 `@MappedSuperclass`를 붙인다.
> 추상 클래스를 권장한다. 이후 알아보겠지만, 직접 사용할 일이 없기 때문이다.
```java
@Entity  
public class Member extends BaseEntity {
}
```
공통 속성을 가질 클래스에서 상속을 한다.
>`extends` 키워드를 사용하기에 자바에서는 상속이라고 표현하지만, 테이블 측면에서 `@MappedSuperclass`는 상속이 아니다.
```sql
create table Member (
	//...
	createdBy varchar(255),
	createdDate timestamp(6),
	lastModifiedBy varchar(255),
	lastModifiedDate timestamp(6),
	primary key (MEMBER_ID)
)
```
결과적으로 공통 속성을 가지는 테이블이 `@MappedSuperclass` 클래스의 필드들이 포함되어 생성되었다.

즉, `@MapedSuperclass`는 엔티티가 아니며 테이블과 매핑이 되지도 않는다.
부모 클래스를 상속 받는 자식 클래스에 매핑 정보만 제공한 것이다.
따라서 조회, 검색이 불가능하다.
