## 1. CASCADE
특정 엔티티를 영속 상태로 만들 때 연관된 엔티티도 함께 영속 상태로 만들고 싶을 때 사용한다.
이 옵션을 사용할 경우, 부모 엔티티를 저장할 때 자식 엔티티도 함께 저장할 수 있다.

```java
@Entity  
public class Parent {  
  
    @Id  
    @GeneratedValue    
    private Long id;  
  
    private String name;  
  
    @OneToMany(mappedBy = "parent")  
    private List<Child> children = new ArrayList<>();
}

@Entity  
public class Child {  
  
    @Id  
    @GeneratedValue    
    private Long id;  
  
    private String name;  
  
    @ManyToOne  
    @JoinColumn(name = "parent_id")  
    private Parent parent;
}
```
위와 같이 parent와 child가 부모-자식 관계를 가지고 있다고 하자.

parent, child를 저장하려면 어떻게 해야 할까?
```java
Child child1 = new Child();  
Child child2 = new Child();  
  
Parent parent = new Parent();  
parent.addChild(child1);  
parent.addChild(child2);  
  
em.persist(parent);  
em.persist(child1);  
em.persist(child2);
```
모든 child, parent를 만들고 모두 각각 `em.persist()`를 해주어야 한다.
만약 child를 영속화하지 않을 경우 insert문이 발생하지 않는다.

```java
@OneToMany(mappedBy = "parent", cascade = CascadeType.ALL)  
private List<Child> children = new ArrayList<>();
```
parent의 children 매핑 관계에 cascade type을 ALL로 주어 설정했다.
```java
Child child1 = new Child();  
Child child2 = new Child();  
  
Parent parent = new Parent();  
parent.addChild(child1);  
parent.addChild(child2);  
  
em.persist(parent);  
```
그리고 child를 명시적으로 영속화하지 않았다.
```sql
Hibernate: 
insert 
    into
        Parent (name, id) 
    values
        (?, ?)
Hibernate: 
insert 
    into
        Child (name, parent_id, id) 
    values
        (?, ?, ?)
Hibernate: 
insert 
    into
        Child (name, parent_id, id) 
    values
        (?, ?, ?)
```
하지만 옵션을 주지 않았을 경우와 다르게 children도 모두 insert sql이 발생한다.
### 종류
- ALL : 모두 적용
- PERSIST : 영속
- REMOVE : 삭제
- MERGE : 병합
- REFRESH
- DETACH
## 2. orphanRemoval
고아 객체를 제거하는 옵션이다.
고아 객체란 부모 엔티티와 연관관계가 끊어진 자식을 의미한다.

`orphanRemoval`은 고아 객체를 삭제하는 기능을 제공한다.
```java
@OneToMany(mappedBy = "parent", cascade = CascadeType.ALL, orphanRemoval = true)  
private List<Child> children = new ArrayList<>();
```
위의 parent-child 예시에서 부모 연관관계 속성에 `orphanRemoval=true`를 설정했다.
```java
Parent findParent = em.find(Parent.class, parent.getId());  
findParent.getChildren().remove(0);
```
그리고 부모 객체에서 조회한 자식 객체를 삭제한다.
```sql
delete 
    from
        Child 
    where
        id=?
```
이 때 삭제한 자식 객체를 제거하기 위한 delete 쿼리가 발생한다.

만약, `orphanRemoval=true`인 상황에서 부모 객체를 삭제하면 어떻게 될까?
```sql
Hibernate: 
delete 
    from
        Child 
    where
        id=?
Hibernate: 
delete 
    from
        Child 
    where
        id=?
Hibernate: 
delete 
    from
        Parent 
    where
        id=?
```
모든 자식 객체와 그 부모 객체가 삭제하는 sql이 발생한다.
이 경우, `CascadeType.REMOVE`와 동일하게 동작한다.
