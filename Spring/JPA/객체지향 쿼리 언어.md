## 1. 객체지향 쿼리 언어란?
데이터베이스 테이블을 대상으로 하는 데이터 중심의 쿼리(SQL)가 아닌 엔티티 객체를 대상으로 하는 쿼리이다.
JPA는 이 객체지향 쿼리 언어를 분석한 다음 적절한 SQL을 만들어 데이터베이스를 조회한다. 그리고 조회한 결과로 엔티티 객체를 생성해서 반환한다.
## 2. 종류
### 1️⃣ [[JPQL]]
JPA가 SQL을 추상화하여 특정 데이터베이스 SQL에 의존하지 않는 쿼리를 제공한다.
```java
List<Member> result = em.createQuery("select m from Member m where m.username like '%kim%'", Member.class).getResultList();
```
실제 수행된 SQL은 아래와 같다.
```sql
select
	m1_0.MEMBER_ID,
	m1_0.USERNAME,
from
	Member m1_0 
where
	m1_0.USERNAME like '%kim%' escape ''
```

JPQL은 SQL보다 간단하고 객체지향이므로 편리하게 사용할 수 있지만,
문자기반 쿼리이기 때문에 오탈자를 컴파일 시점에 잡아낼 수 없다는 단점이 있다.
### 2️⃣ Criteria
JPQL을 편하게 작성하도록 도와주는 API, 빌더 클래스 모음이다.
JPQL의 단점인 문자 기반 쿼리에서 나아가 쿼리를 코드로 작성할 수 있다.
```java
//Criteria 사용 준비  
CriteriaBuilder cb = em.getCriteriaBuilder();  
CriteriaQuery<Member> query = cb.createQuery(Member.class);  
//루트 클래스 (조회를 시작할 클래스)  
Root<Member> m = query.from(Member.class);  
//쿼리 생성  
CriteriaQuery<Member> cq = query.select(m).where(cb.equal(m.get("username"), "kim"));  
List<Member> resultList = em.createQuery(cq).getResultList();
```
실제 수행된 SQL은 아래와 같다.
```sql
select
	m1_0.MEMBER_ID,
	m1_0.USERNAME,
from
	Member m1_0 
where
	m1_0.USERNAME=?
```

Criteria는 컴파일 시점에 오류를 발견할 수 있고, 동적 쿼리를 작성하기 편하다.
하지만 코드로 작성하는 부분이 너무나도 복잡하고 장황하여 사용이 어렵다.
### 3️⃣ QueryDSL
Criteria 쿼리처럼 JPQL을 편하게 작성하도록 도와주는 빌더 클래스 모음이다.
Criteria와 다르게 비표준 오픈소스 프레임워크이다.
```java
JPAFactoryQuery query = new JPAQueryFactory(em);
QMember m = QMember._member_;
List<Member> list = query.selectFrom(m)
			.where(m.age.gt(18))
			.orderBy(m.name.desc())
			.fetch();
```
QueryDSL 또한 코드 기반이기에 컴파일 시점에 문법 오류를 찾을 수 있고 동적쿼리 작성이 편리하다.
그리고 단순하고 쉽게 작성할 수 있어 실무 사용을 권장한다.
### 4️⃣ 네이티브 SQL
JPA가 제공하는 SQL을 직접 사용하는 기능이다.
JPQL로 해결할 수 없는 특정 데이터베이스에 의존적인 기능을 이용해야 할 시 사용한다.
```java
List<Member> resultList = em.createNativeQuery("select member_id, username from member").getResultList();
```
실제 수행된 SQL은 아래와 같다.
```sql
select
	member_id,
	username 
from
	member
```
